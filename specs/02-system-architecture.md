# System Architecture

## Structural Elements

### Notes (Events)
- Time-bound
- Contextual
- Subjective
- Immutable at the raw layer

Examples:
- reflections
- thoughts
- questions
- source impressions

### Hub Notes (Places)
- Persistent conceptual gathering points
- Non-authoritative
- Allowed to contain contradictions
- Accumulate links over time

Examples:
- Technology and Emotion
- Identity Change
- Electronic Music
- Caregiving

---

## Layered Model

### Capture Layer
Purpose: speed and fidelity

- Mobile capture
- Desktop capture
- Quick or deliberate modes

### Processing Layer
Purpose: contextualization and connection

- Theme extraction
- Hub suggestion
- Link suggestion
- Temporal framing

### Knowledge Layer
Purpose: durability and navigation

- Markdown notes
- Obsidian-style links
- Hub-based traversal

---

## Capture Equivalence Principle
No capture surface or note type is more authoritative than another.
Meaning emerges from accumulation and connection, not origin.

## Structural Rule
Notes link outward to hubs.
Hubs link inward to notes.
