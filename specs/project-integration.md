# Project Integration

## Purpose

Projects represent active lines of inquiry or creation.
They are temporary, goal-oriented lenses applied to the archive.

Projects do not own notes.
Notes may outlive projects.

---

## Relationship to Notes

- A project may reference notes
- A project may link to hubs
- Notes should not be rewritten to serve a project

Projects pull from the archive.
They do not reshape it.

---

## Lifecycle

1. Project begins
2. Notes are referenced or created
3. Insights accumulate
4. Project concludes or pauses
5. Project is archived

The notes remain.

---

## Automation Guidance

Automation may:
- link notes to projects
- suggest relevant hubs

Automation must not:
- move notes into project folders permanently
- treat project notes as authoritative summaries
